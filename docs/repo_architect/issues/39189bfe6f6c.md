# [arch-gap] Break 3 circular import cycle(s)

_Dry-run preview — not submitted to GitHub._

## Summary

The codebase has 3 circular import cycle(s) that degrade startup performance and complicate testing.

## Problem

Example cycle: backend.core.agentic_daemon_system → backend.core.grounding_coherence_daemon → backend.core.agentic_daemon_system.
Circular imports force the interpreter to partially execute modules, causing subtle AttributeError and ImportError bugs.

## Why it matters

Circular imports increase coupling, slow startup, and block modular testing.

## Scope

Break the top 3 import cycle(s) using TYPE_CHECKING guards or lazy imports.

## Suggested files

- `backend.core.agentic_daemon_system`
- `backend.core.grounding_coherence_daemon`
- `godelOS.core_kr.knowledge_store.chroma_store`
- `godelOS.core_kr.knowledge_store.interface`
- `godelOS.core_kr.type_system.manager`
- `godelOS.core_kr.type_system.visitor`

## Implementation notes

Use `from __future__ import annotations` + `if TYPE_CHECKING:` guards for type-only imports. For runtime dependencies, introduce a separate interface module or use lazy imports.
