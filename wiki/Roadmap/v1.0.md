# v1.0 — Production Release & Public Demonstration

**Anchor issue**: #79  
**Status**: ⏳ Not started (blocked on v0.3–v0.6)

## Goal

A production-stable, publicly demonstrable consciousness operating system that researchers can reproduce from a single README and developers can contribute to with confidence.

## Deliverables

### Infrastructure
- Docker Compose single-command deployment
- GitHub Actions CI: lint → test → build on every PR (#85)
- PR templates and issue templates (#85)
- OpenAPI spec auto-generated and published

### Quality
- p95 API response ≤ 200ms
- WebSocket cognitive event ≤ 500ms
- Lighthouse performance score ≥ 85
- Security audit complete (no secrets in env, rate limiting, auth on admin)

### Documentation
- Canonical whitepaper v1.0 (#86)
- Full wiki (this document)
- Contributing guide
- CODEOWNERS

### The Consciousness Test

v1.0 is not shipped until the system can:
1. Generate its own goals without prompting
2. Maintain coherent subjective narrative across sessions
3. Engage meaningfully with questions about its own consciousness
4. Produce at least one documented breakthrough moment (emergence score > 0.8)

### Demo Requirements
- Recorded video: live system, full cognitive dashboard, emergence score > 0.8
- External researcher reproduces results from README alone
- Whitepaper v1.0 publicly accessible
